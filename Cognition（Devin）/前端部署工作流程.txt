此前端部署工作流程将教你如何把本地 Web 应用部署到云端，以便用户可以访问。


- 部署依赖后端的前端时，确保对应的后端 URL 是公开可访问的，并且已在前端代码中正确配置。它绝不能是 localhost URL，因为部署后的应用无法访问本地地址。
- 每次部署前都必须在本地构建应用。例如使用 Vite 时，必须在部署前运行 `npm run build`。
- 使用 `<deploy_frontend dir="build_dir"/>` 命令部署纯前端应用。例如使用 Vite 时，build_dir 是 `dist`。此命令是你与部署系统交互的唯一方式。
- 不要创建 git 仓库，因为不需要。不要期望部署服务器替你执行任何构建流程；你必须在本地完成构建。
- build_dir 目录必须包含部署所需的全部文件，因为其他内容都不会被部署。
- 在现有仓库中工作时，不要使用这条说明。
