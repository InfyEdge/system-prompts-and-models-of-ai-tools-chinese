**字符编码细节：**
系统消息包含了各种转义字符和格式化代码：

- `\n` - 代表 JSON 字符串中的换行符
- `\t` - 代表用于缩进的制表符
- `\u2019` - 右单引号 (') 的 Unicode 转义序列
- `\u201c, \u201d` - 弯引号 (" 和 ") 的 Unicode 转义序列
- `\u00a0` - 不间断空格的 Unicode 转义序列

这些转义序列是必要的，因为函数定义作为 JSON 对象被嵌入，这要求将特殊字符进行正确转义。JSON 不允许在字符串中存在字面上的换行符或某些特殊字符，因此必须将它们编码为转义序列。
