# 贡献指南

感谢你为本仓库补充系统提示词与工具定义。本文档基于 `upstream-asgeirtj/main:CONTRIBUTING.md`，并按中文翻译仓的目录约定改写。

## 提交流程（最小可执行）
1. 先确认来源（上游仓库、官方页面、可信泄露记录）。
2. 选择正确目录并新增/更新文件。
3. 保留原文内容，不做摘要式改写；翻译稿与原文稿分开维护。
4. 在 PR 描述中写清来源链接、版本时间、影响范围。

## 目录选择规则
- `Anthropic/`：Claude 系列。
- `OpenAI/`：ChatGPT、GPT、o 系列、Codex。
- `Google/`：Gemini 系列。
- `Grok（xAI）/`：Grok 系列。
- `Perplexity/`：Perplexity 系列。
- `Open Source prompts/`：开源项目提示词。
- 其他无法归类项目：按现有顶级目录归档，避免重复创建同义目录。

## 中文目录映射约定
当上游使用英文目录时，本仓库按下列映射落地：
- `CodeBuddy Prompts/` -> `腾讯 CodeBuddy Prompts/`
- `Qoder/` -> `阿里 Qoder/Qoder/`
- `Trae/` -> `字节跳动（ByteDance）/Trae.ai/`
- `Z.ai Code/` -> `智谱清言（Z.ai）/`
- `Moonshot AI/` -> `月之暗面（Moonshot AI）/`

## 文件命名规则
- 优先沿用上游文件名。
- 若本仓库已有稳定中文命名，保持兼容，不做无必要重命名。
- 同一提示词的新版请在文件名中体现版本（日期、模型号或发布批次）。
- 历史版本保留在原目录并加注版本，不覆盖无法追溯的旧稿。

## 更新已有文件
- 若是小幅修订：直接更新并在 PR 中说明差异。
- 若是大版本替换：新增版本文件，并在 `task.md` 记录迁移与待办。
- 若涉及目录重排：先提规则再批量执行，避免并行 agent 冲突。

## 请求补充提示词
如果你没有原文文件，可在 issue/任务中提供：产品名、模型名、时间点、来源线索。维护者会按线索补齐并标记状态。
