# Open Source Prompts 归档整理计划

## 当前问题
1. "Open Source prompts/" 目录中的内容与主目录存在重复
2. 一些工具已有独立的公司目录（如 Cline、Bolt.new）
3. 目录结构不够统一

## 归档方案

### 方案A：合并到对应公司目录（推荐）
- Bolt → StackBlitz（Bolt.new）/
- Cline → Cline（VS Code插件）/
- Gemini CLI → Google（Gemini）/
- Codex CLI → 保留或归档到 Tools/
- Lumo → Proton（Lumo）/
- RooCode → 新建 RooCode/ 或归档
- OpenCode → 归档到 Tools/Open-Source/

### 方案B：重命名为 Open-Source-Tools/
保持独立，但改为更规范的目录名

### 方案C：创建 Archive/ 目录
将历史版本或重复内容移入归档

## 建议操作步骤

1. **检查重复内容**
   ```bash
   # 检查 Cline
   diff "Open Source prompts/Cline/" "Cline（VS Code插件）/"
   
   # 检查 Bolt
   diff "Open Source prompts/Bolt/" "StackBlitz（Bolt.new）/"
   ```

2. **合并非重复内容**
   - 如果 Open Source prompts/ 中有新版本，合并到主目录
   - 如果完全重复，删除 Open Source prompts/ 中的版本

3. **处理独立项目**
   - RooCode、OpenCode 等创建独立目录或归档

4. **更新 README**
   - 移除 "Open Source prompts/" 章节
   - 更新目录结构说明
   - 添加完整的翻译进度说明

## 执行建议
优先采用**方案A**，将内容合并到对应公司目录，保持结构统一。
